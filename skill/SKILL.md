# hwpx 계약 서식 자동완성 스킬

## 개요

계약서를 인풋으로 받아, 5종 서식 템플릿의 플레이스홀더(`{{1}}`~`{{13}}`)를 자동으로 채워
완성된 hwpx 파일을 제공한다.

---

## 템플릿 파일 위치

스킬 디렉토리의 `templates/` 폴더에 5종 템플릿이 번들되어 있다.

| 파일명 | 서식명 | 사용 플레이스홀더 |
|--------|--------|----------------|
| `1__계약_시_구비서류.hwpx` | 계약 시 구비서류 | {{1}}~{{13}} 전체 |
| `2__착수계.hwpx` | 착수계 | {{1}}{{2}}{{3}}{{5}}{{6}}{{7}}{{9}}{{10}} |
| `3__선금신청서.hwpx` | 선금신청서 | {{1}}{{2}}{{3}}{{5}}{{6}}{{7}}{{9}}{{10}} + `_seongeum_rate` |
| `4__완료계.hwpx` | 완료계 | {{1}}{{2}}{{3}}{{5}}{{6}}{{7}}{{9}}{{10}} |
| `5__청구서.hwpx` | 청구서 | {{1}}{{2}}{{3}}{{5}}{{6}}{{7}}{{9}}{{10}} + `_cheonggu_amount` |

---

## 플레이스홀더 의미표

| 번호 | 의미 | 비고 |
|------|------|------|
| {{1}} | 용역수행업체명 | |
| {{2}} | 용역수행업체 소재지 | |
| {{3}} | 용역수행업체 대표자 | |
| {{4}} | 용역수행업체 업종 | 구비서류에만 사용 |
| {{5}} | 계약일자 | 형식: YYYY. MM. DD. (끝 온점 포함) |
| {{6}} | 용역명 | |
| {{7}} | 계약금액 (계약 총액) | 예: 일금 이천팔만원정 (₩20,080,000) — 청구서에서도 계약 총액 그대로 입력 |
| {{8}} | 계약보증금액 | 구비서류에만 사용 |
| {{9}} | 계약시작일 (착수일) | 형식: YYYY. MM. DD. (끝 온점 포함) |
| {{10}} | 계약종료일 | 형식: YYYY. MM. DD. (끝 온점 포함) |
| {{11}} | 보증 시작일 | 계약종료일 +1일 (자동계산) |
| {{12}} | 보증 종료일 | 보증시작일 +364일 (자동계산) |
| {{13}} | 발주부서 | 구비서류에만 사용 |
| `_cheonggu_amount` | 청구금액 (청구서 전용) | 정수 입력 시 한글 금액 자동 변환. 예: `10040000` → `금 일천사만원정 (￦ 10,040,000)` |
| `_seongeum_rate` | 선금 신청비율 (선금신청서 전용) | 정수(%) 입력. 신청금액은 {{7}} × 비율로 자동 계산 |
| `_submission_year` | 제출일 연도 (전 양식 공통, 선택) | 미지정 시 {{10}}(계약종료일) 연도 → 오늘 연도 순으로 자동 결정. 템플릿의 `YYYY.   .    .` 부분의 연도 4자리만 치환 |

> **자동 계산**: {{11}}(보증시작일)과 {{12}}(보증종료일)은 {{10}}(계약종료일) 기반으로 자동 계산됨.
> **업종 자동설정**: 업체명이 **팜앤디 협동조합** 또는 **청춘작당 협동조합**인 경우 {{4}} 업종은 `학술연구용역, 디자인, 행사`로 자동 설정됨.
> **소재지 자동처리**: {{2}} 소재지는 우편번호를 자동 제거하며, 좁은 표 셀(구비서류 등)에서는 시군구/읍면동 기준으로 자동 두 줄 분리됨.
> **계좌 자동설정**: 청구서 작성 시 업체명 기반으로 은행명·계좌번호가 자동 입력됨.
> - 팜앤디 협동조합: 농협 / 351-4022-2449-23
> - 청춘작당 협동조합: 농협 / 351-1068-2100-13
> **통장사본 자동첨부**: 청구서 작성 시 업체명에 맞는 통장사본을 별도 PDF 파일로 함께 생성함. hwpx와 PDF 두 파일을 함께 제출하면 됨.
> **청구서 계약금액 vs 청구금액 분리**: 청구서의 "2. 계약금액"은 항상 계약 총액({{7}})이며, "5. 청구금액"은 별도 `_cheonggu_amount` 키로 입력한다. 분할 청구(선금/잔금 등) 시 두 금액이 다를 수 있으므로 절대 혼동하지 말 것.
> **제출일 연도 자동 치환**: 템플릿의 제출일 `YYYY.   .    .` 부분에 박혀 있는 기본 연도(2025)는 자동으로 실제 제출 연도로 교체된다. 일·월은 사용자가 직접 기입하도록 비워둠. 우선순위: `_submission_year` > {{10}} 연도 > 오늘 연도.

## 워크플로우

### Step 1: 작성할 서식 확인
사용자가 원하는 서식을 명시하지 않은 경우 확인한다.
- 계약 시 구비서류 / 착수계 / 선금신청서 / 완료계 / 청구서 (복수 선택 가능)

### Step 2: 계약서 요청 및 정보 추출
계약서 파일(hwpx, PDF, 이미지 등)을 요청한다.
계약서에서 아래 항목을 추출한다:
- 업체명, 소재지, 대표자, 업종
- 계약일자, 용역명, 계약금액, 계약보증금액
- 착수일, 계약종료일, 발주부서

**추출 후 반드시 사용자에게 확인**: 아래 형식으로 정리하여 검토 요청
```
추출된 계약 정보를 확인해 주세요:
- 업체명({{1}}): ...
- 소재지({{2}}): ...
- 대표자({{3}}): ...
...
```

**청구서 작성 시 추가 확인**: 청구금액이 계약 총액과 다를 수 있으므로(선금/잔금/분할 청구 등) 사용자에게 청구금액을 명시적으로 확인한다.
- 청구금액 == 계약 총액 → `_cheonggu_amount`에 {{7}}의 숫자값을 그대로 입력
- 청구금액 != 계약 총액(분할 청구 등) → 사용자가 지정한 청구금액을 `_cheonggu_amount`에 입력

### Step 3: hwpx 파일 생성
사용자 확인 후 스크립트로 파일 생성:

```bash
python3 {스킬경로}/scripts/fill_hwpx.py \
  "{스킬경로}/templates/{템플릿파일}.hwpx" \
  "/mnt/user-data/outputs/{출력파일명}.hwpx" \
  '{"1":"업체명","2":"소재지",...,"_cheonggu_amount":10040000}'
```

복수 서식 요청 시 파일별로 반복 실행한다.

### Step 4: 파일 제공 및 검토 요청
`present_files` 도구로 완성된 hwpx 파일을 제공하고, 검토를 요청한다.
수정이 필요한 경우 Step 2로 돌아가 값을 수정 후 재생성한다.

## 입력값 형식 규칙 (반드시 준수)

### 금액 형식
- 한글 금액 뒤에는 반드시 **"정"** 을 붙인다
- 한글 금액의 천/백/십 자리에 1이 올 때도 **"일"을 생략하지 않는다** (회계·계약 문서 규약)
  - 올바른 예: `일천사만`, `일천팔백팔십만`, `일십만`
  - 잘못된 예: `천사만`, `천팔백팔십만`, `십만`
- 올바른 예: `일금 일천팔백팔십만원정 (₩18,800,000)`, `금 일천사만원정 (￦ 10,040,000)`
- 잘못된 예: `일금 일천팔백팔십만원 (₩18,800,000)`, `금 천사만원정 (￦ 10,040,000)`

### 날짜 형식
- 모든 날짜 끝에는 반드시 **온점(.)** 을 붙인다
- 올바른 예: `2026. 04. 06.`
- 잘못된 예: `2026. 04. 06`
- 적용 항목: {{5}} 계약일자, {{9}} 착수일, {{10}} 계약종료일, {{11}} 보증시작일, {{12}} 보증종료일

### 글자 간격 / 줄바꿈
- 입력값에 임의로 공백을 줄이거나 한 줄로 압축하지 않는다
- 템플릿 원본의 문자 간격과 줄 설정을 그대로 유지한다
- 긴 내용은 템플릿 셀/칸 안에서 자연스럽게 줄바꿈되도록 그대로 입력한다

## 예외 처리

| 상황 | 대응 |
|------|------|
| 계약서에서 일부 정보 추출 불가 | 해당 항목 `[확인 필요]`로 표시 후 사용자에게 직접 입력 요청 |
| 날짜 형식이 다양한 경우 | YYYY. MM. DD 형식으로 통일하여 입력 |
| 복수 서식 동시 요청 | 파일별로 순서대로 생성하여 한번에 제공 |
| 보증일 직접 입력 요청 | {{11}}, {{12}} 자동계산 대신 사용자 입력값 사용 |
| 완료계 완료일 | 기본값은 계약종료일({{10}})로 자동 채워짐. 사용자가 별도 일자(예: 일자 빈칸)를 요청하면 생성 후 section.xml의 해당 문자열을 직접 치환 |
| 청구서 청구금액 | {{7}}(계약 총액)과 별도. `_cheonggu_amount`로 분리 입력. 미입력 시 빈칸 유지 |
| 선금신청서 | 작성 전 반드시 신청비율(%)을 사용자에게 확인 후 진행. 신청금액은 계약금액 × 비율로 자동 계산 (만원 이하 절삭). 사용계획 항목은 담당자가 직접 기입하도록 비워둠 |
